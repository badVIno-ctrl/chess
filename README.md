# Шахматы — GitHub Pages

## Файлы

- `index.html` — основной файл сайта
- `rules.html` — страница с правилами
- `helper-worker.js` — фоновый расчёт ходов ИИ
- `helper.html` — резервная страница помощника
- `chess.html` — совместимый редирект на `index.html`
- `.nojekyll` — отключает Jekyll-обработку на GitHub Pages

промпт:
 Ты — Senior Full-Stack разработчик и эксперт по WebGL (Three.js), создающий премиальные браузерные игры.
Задача: Написать полноценную браузерную 3D-игру «Шахматы» с интегрированным ИИ-соперником и чатом, а также отдельную масштабную страницу с обучением. Весь проект должен состоять строго из двух файлов: chess.html (игра, логика, стили) и rules.html (подробное руководство). Весь интерфейс, чат ИИ и правила должны быть исключительно на русском языке.
1. Архитектура и Файловая структура:
chess.html: Весь HTML, CSS (в теге <style>) и JavaScript (в теге <script>) для самой игры.
rules.html: Весь код для интерактивного и подробного руководства по игре.
Библиотеки (Three.js для графики, любые вспомогательные для физики/анимации) подключать исключительно через CDN.

2. Визуальный стиль и Дизайн (UI/UX):
Тематика: Роскошный матовый черный и золото. Полностью исключить синие, фиолетовые и розовые цвета.
Фигуры игрока (светлые): Высокополированное золото с теплым золотым свечением.
Фигуры противника (темные): Глубокий матовый черный, покрытые серебристым сиянием для контраста и выделения.
Доска (клетки): Чередование глубокого матового черного и полированного темно-золотого/черного.
Освещение: Насыщенное, теплое золотое свечение. Камера под небольшим углом сверху.
UI и Чат: Эффект glassmorphism (backdrop-filter: blur()). Фон верхней панели и панели чата (справа) — матовый черный. Весь текст интерфейса (Сложность, Легко, Стандарт, Сложно, Ваш ход, Отменить ход, Новая игра) — золотого цвета. Активная кнопка сложности — сплошная золотая пластина с черным текстом.

3. Игровая механика, 3D и Управление:
Движок: Three.js. Шахматы и доска генерируются процедурно или используются стилизованные базовые 3D-примитивы.
Управление: Drag & Drop или «клик по фигуре -> клик по пустой клетке».
Камера: Добавить возможность приближать и отдалять доску колесом мыши (scroll) или щипком (pinch).
Ход назад: Добавить кнопку «Отменить ход» (Undo), чтобы игрок мог вернуть свою и вражескую фигуру на предыдущую позицию.
Адаптивность: 100% оптимизация под мобильные версии. Поддержка touch-событий. На телефонах чат должен сворачиваться или накладываться поверх экрана с сильным блюром.
Брендинг: В левом нижнем углу — минималистичная золотая кнопка «viora studio». При клике открывается ссылка: https://t.me/VioraStudio (в новой вкладке).

4. Правила игры в Шахматы (строгая реализация логики):
Движок должен строго соблюдать все правила шахмат:
Уникальные паттерны хода для Пешки, Ладьи, Коня, Слона, Ферзя и Короля.
Взятие чужих фигур (по одной за ход).
Спец. ходы: Рокировка (короткая и длинная), Взятие на проходе (En passant), Превращение пешки (Promotion — по умолчанию в Ферзя или с выбором).
Состояния: Определение Шаха, Мата (конец игры) и Пата (ничья).
Запрет на ходы, которые подставляют собственного короля под шах.

5. Интеграция с ИИ (Mistral API) и Чат:
Модель: mistral-tiny.
Ключи API: Основной, Запасной. Автоматическое переключение на запасной ключ при падении/ошибке основного.
Сложность: 3 режима (Легкий, Стандарт, Сложный). Влияет на глубину просчета ходов и системный промт ИИ.
Чат: Когда ИИ делает ход, фигура плавно анимируется, и в чате появляется сообщение от ИИ с комментариями по ситуации на доске.
Очистка: Написать JS-функцию с RegExp, которая очищает ответ ИИ от Markdown-символов (звездочки **, решетки #), чтобы в чат попадал только чистый текст.

6. Сохранения и Обучение (rules.html):
Сохранения (localStorage): Сохранять положение на доске, историю чата и выбранную сложность. При обновлении chess.html игра продолжается с того же места.
Первый вход: При отсутствии сохранений поверх экрана появляется красивый золотой pop-up: «Умеете играть в шахматы? [Да / Нет]».
Если «Да» — начинается игра.
Если «Нет» — перенаправление на файл rules.html.
Содержимое rules.html: Огромный, подробно расписанный лендинг с правилами.
Оформление в том же стиле: матовый черный и золото.
Разделы: Цель игры, ценность фигур, как ходит каждая фигура (с визуальными схемами на CSS/HTML), особые правила (рокировка, взятие на проходе, превращение), что такое шах и мат.
В конце огромная золотая кнопка «Я готов играть!», возвращающая в chess.html.
Конечный результат: Выдай мне только два полностью рабочих и отформатированных блока кода: один для chess.html, второй для rules.html. Убедись, что код не обрывается и содержит полную реализацию описанных функций.
